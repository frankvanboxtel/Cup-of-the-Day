# Cup of the Day

Minimal Node.js + TypeScript project for working with the Cup of the Day CSV dataset in this repository.

## Requirements

- Node.js 20+
- npm

## Install

```bash
npm install
```

## Scripts

- `npm run generate` reads all COTD CSV files from `data/source-csvs/` and writes per-cup JSON files to `data/generated-jsons/`
- `npm run validate:results` checks that the generated result files contain a complete event-number sequence with no gaps
- `npm run generate:all` runs the full generation pipeline: results JSON, result validation, alias proposals/generated aliases, and HTML output
- `npm run generate:aliases` reads the generated JSON files, proposes player alias additions in `data/generated-jsons/player-alias-proposals.json`, and writes generated alias groups to `data/player-settings/player-aliases.generated.json`
- `npm run generate:html` reads the JSON files in `data/generated-jsons/`, combines `data/player-settings/player-aliases.json` with `data/player-settings/player-aliases.generated.json`, applies `data/player-settings/display-only-names.json`, and builds a static HTML site in `html/`

## Project Structure

```text
data/
  source-csvs/
    CSV source files
  generated-jsons/
    Generated per-cup JSON files
    Generated alias proposal output
  player-settings/
    player-aliases.json
    player-aliases.generated.json
    display-only-names.json
html/
  Generated static HTML pages
src/
  generate-results.ts
  validate-results.ts
  generate-alias-proposals.ts
  generate-html.ts
dist/
  Compiled JavaScript output after build
```

## Current Behavior

Result generation detects event blocks dynamically from the CSV headers, which makes the importer resilient to column drift and `COTD`/`COTW` naming differences. The validation step checks that generated result files cover a complete event-number sequence.

## Data Notes

- The CSV files are stored in `data/source-csvs/`
- Generated event JSON files are stored in `data/generated-jsons/`
- `data/player-settings/player-aliases.json` is for manually curated aliases
- `data/player-settings/player-aliases.generated.json` is generated by `npm run generate:aliases`
- `data/player-settings/display-only-names.json` is for result labels that should stay visible as text but should not get player pages or links
- HTML generation merges the manual and generated alias files into shared driver and author pages
- The TypeScript configuration includes `src/**/*.ts`

## Example Workflow

```bash
npm install
npm run generate:all
npm run dev
npm run build
npm start
```
